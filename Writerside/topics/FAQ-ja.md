# FAQ

%product% を使うべきか、どう使うかといった判断に関する質問をまとめています。「壊れている、どうやって直す?」という質問については、[トラブルシューティング](Troubleshooting-ja.md)を参照してください。

## このプラグインは Azure DevOps Server(オンプレミス)をサポートしていますか?

はい。アカウントを作成する際にサーバーの URL を追加してください。Azure DevOps Server 2019 以降と Azure DevOps Services(クラウド)の両方がサポートされています。

クラウド版は `dev.azure.com/<org>` を使用します。オンプレミス版では、ご自身のサーバー URL(例: `https://devops.example.com`)を使用します。PAT 認証はどちらでも動作します。Microsoft Entra 経由の OAuth は**クラウド専用**です。

## Azure Pipelines をサポートしていますか?

はい。専用の **Pipelines** ツールウィンドウで、パイプラインと実行を閲覧したり、インタラクティブなステージグラフを見たり、色分けされたステップログを読んだり、手動承認ゲートを承認または却下したりできます。これらはすべて IDE 内で完結します。プルリクエストの CI チェックが Azure ビルドを指している場合、**Details…** をクリックすると、その実行がブラウザではなく IDE 内で開きます。

デフォルトでオンになっています。<ui-path>Settings | Version Control | Azure DevOps</ui-path> の **Enable Pipelines integration** で切り替えられます。オフにすると、ツールウィンドウは非表示になり、パイプライン通知やバッジは発生せず、パイプラインチェックはブラウザで開きます。[パイプライン](Pipelines-ja.md)を参照してください。

## 従来の TFVC(Git 以外)リポジトリで動作しますか?

いいえ。このプラグインは Git ベースの Azure Repos 専用です。**TFVC**(Team Foundation Version Control、Azure DevOps における Git の前身であるマイクロソフトの集中型 VCS)はサポートされていません。

## Azure DevOps アカウントなしで使えますか?

いいえ。このプラグインは Azure DevOps 専用です。GitHub や GitLab については、それぞれ同梱の GitHub プラグインまたは JetBrains GitLab プラグインを使用してください。

## 同じ IDE に GitHub と Azure DevOps の両方の PR プラグインを入れられますか?

はい。それぞれが独立したツールウィンドウを登録し、状態を共有しません。単一のプロジェクトが GitHub と Azure DevOps の両方を指すリモートを持っている場合、両方のツールウィンドウが表示され、それぞれが自身のリモートの PR のみを一覧表示します。

## OAuth と PAT のどちらを選ぶべきですか?

| 使うべきもの…    | 状況                                                                                                          |
|-----------------|--------------------------------------------------------------------------------------------------------------|
| **OAuth**       | クラウド版(`dev.azure.com`)を使用していて、組織が OAuth を禁止しておらず、MFA プロンプトをインラインで表示したい場合。 |
| **PAT**         | Azure DevOps Server(オンプレミス)を使用している、組織のポリシーで PAT が必須、または OAuth ハンドラーの問題に遭遇した場合。|

OAuth トークンは自動的に更新されます。PAT は作成時に設定した日付で期限切れになります。PAT は設計上 MFA をバイパスします。PAT を生成するにはユーザーが既に認証済みである必要がありますが、トークン自体は再度プロンプトを表示しません。

## 私のコードは Azure DevOps 以外の場所に送信されますか?

AI 機能を明示的に有効化してプロバイダーを設定していない限り、送信されません。機能ごとの完全なデータフローについては、[プライバシーとデータ](Privacy-and-Data-ja.md)を参照してください。

AI が無効(マスタースイッチがオフ)の場合、プラグインは Azure DevOps 組織以外への送信を一切行いません。

## AI 呼び出しを一切行わずにプラグインを使うにはどうすればよいですか?

<ui-path>Settings | Version Control | Azure DevOps | AI Settings</ui-path> の上部にある **Enable AI assistance** のチェックを外してください。すべての AI アフォーダンスがメニューやツールバーから消え、AI 呼び出しは行われません。

また、AI をオンのままにして、すべての機能をローカルの **Ollama** インスタンスにルーティングし、完全にオンデバイスで推論を行うこともできます。

## PR のメトリクスはどこで確認できますか?

[統計](Statistics-ja.md)タブには、KPI とチャート(マージまでの時間、レビュー速度、投票の分布など)が表示されます。これらはキャッシュされたデータからローカルで計算されます。これは表示専用のダッシュボードです。エクスポート可能な組織全体のレポートには Azure DevOps Analytics を使用してください。

## メモリと CPU のフットプリントはどのくらいですか?

このプラグインは、IDE に同梱された `collaboration-tools` ツールキット(GitHub プラグインが使用しているのと同じツールキット)上の薄い Swing UI です。アイドル時には、IDE のベースラインに加えて約 30〜60 MB のヒープを追加します。アクティブな同期(PR リスト、コメント、差分の取得)では、さらに数 MB が追加されます。

巨大な組織(多数のリポジトリにまたがる 1000 以上の PR)の場合、PR リストは仮想化され、表示されている行のみが UI コンポーネントを保持します。

## プラグインは何かを匿名でアップロードしますか? テレメトリはありますか?

いいえ。出荷版はテレメトリ、クラッシュレポート、使用状況分析を収集しません。唯一の送信は Azure DevOps 組織と(AI が有効な場合)設定した AI プロバイダーへのものです。[プライバシーとデータ](Privacy-and-Data-ja.md)を参照してください。

## Azure Repos Wiki をサポートしていますか?

いいえ。このプラグインはプルリクエストに範囲を限定しています。Wiki の編集には Azure DevOps の Web UI を使用してください。
