# 権限

%product% が表示できる内容は、資格情報が読み取りを許可されている範囲によって決まります。ここでは全体像を示します。省略できない権限が 1 つ、追加機能を静かに解放する権限が 5 つあります。

> **手早いレシピ:** PAT を作成するときは **Full access** を選ぶか、Microsoft でサインインするときは **Full access** の階層を選んでください。これで下記のすべてが問題なく動作します。
> {style="tip"}

## 省略できない権限

**Code (Read &amp; write)** はプラグインの基盤です。プルリクエストの一覧表示、差分の読み取り、コメント、投票、完了処理を担います。この権限がないトークンでは有用な操作が一切できないため、ログインダイアログはその場で拒否し、追加すべきスコープを知らせるメッセージを表示します。空で壊れたツールウィンドウにサインインした状態になることは決してありません。

## それ以外はすべて任意

これらはいずれも体験を豊かにします。欠けている場合は、その機能が単にオフになるだけです。エラーも壊れたパネルもありません。

| 権限 (PAT スコープ)              | 解放される機能                                                                                              | ない場合                                                                                          |
|---------------------------------|------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------|
| **Code → Status**               | **Checks** パネル。PR のビルド/ポリシー結果。                                                               | Checks を読み込めません。PR ビューのその他の部分には影響ありません。                              |
| **User Profile → Read**         | あらゆる場所で実際のプロフィール写真を表示。                                                                | アバターはイニシャルで表示されます。                                                              |
| **Identity → Read**             | @メンションのオートコンプリート、メンションの名前解決、既定ビューの背後にあるチームメンバーシップの検索。   | メンション検索は何も返しません。チームに割り当てられた PR が既定の一覧から消えることがあります。  |
| **Project and Team → Read**     | 既定の PR 一覧の **assigned to my team** 部分のためのチームメンバーシップ。                                 | プラグインは同じ情報について Identity → Read にフォールバックします。                             |
| **Work Items → Read**           | PR 詳細のリンクされた作業項目と Work Items フィルター。                                                     | 作業項目セクションは空のままになります。                                                          |
| **Security → Manage**           | Complete ダイアログの **Override branch policies** オプション。                                             | このオプションは非表示になります。                                                                |

## 欠けている権限の挙動

- **必須の権限が欠けている** - サインインがブロックされ、必要なスコープがログインダイアログに正確に示されます。
- **任意の権限が欠けている** - その機能は静かに劣化し(写真の代わりにイニシャル、空のセクションなど)、それ以外はすべて動作し続けます。
- **チームに割り当てられた PR が利用できない** - これは既定の PR 一覧の表示内容を変えるため、追加すべき権限を説明する一度限りの通知が表示されます。既定の「自分の」ビューの仕組みについては [プルリクエスト](Pull-Requests-ja.md) を参照してください。

## 欠けている権限を修正する

権限は作成時にトークンへ焼き込まれます。プラグインから既存のトークンに追加することはできません。

<procedure title="より多くの権限で再認証する">
    <step><b>PAT:</b> 欠けているスコープを付与した新しいトークンを作成し、<ui-path>Settings | Version Control | Azure DevOps</ui-path> → 鉛筆アイコン → 貼り付けます。</step>
    <step><b>Microsoft (OAuth):</b> アカウントを削除して再度サインインし、権限選択画面で <b>Full access</b> を選びます。</step>
</procedure>

サインインの詳しい手順については [認証](Authentication-ja.md) を参照してください。
