# 권한

%product%가 표시할 수 있는 내용은 자격 증명이 무엇을 읽을 수 있도록 허용되어 있는지에 따라 달라집니다. 여기 전체 지도가 있습니다 - 건너뛸 수 없는 권한 하나, 그리고 부가 기능을 조용히 열어 주는 다섯 개입니다.

> **빠른 방법:** PAT를 만들 때 **Full access**를 선택하거나 Microsoft로 로그인할 때 **Full access** 등급을 선택하세요 - 아래의 모든 것이 그냥 작동합니다.
> {style="tip"}

## 건너뛸 수 없는 하나

**Code (Read &amp; write)**는 플러그인의 토대입니다 - 풀 리퀘스트 나열, diff(변경 내용) 읽기, 댓글 달기, 투표, 완료 처리를 담당합니다. 이 권한이 없는 토큰은 유용한 작업을 아무것도 할 수 없으므로, 로그인 대화 상자가 그 자리에서 거부하고 어떤 범위를 추가해야 하는지 알려 주는 메시지를 표시합니다. 비어 있고 망가진 도구 창에 로그인된 상태로 끝나는 일은 절대 없습니다.

## 나머지는 모두 선택 사항

이들 각각은 경험을 풍부하게 해 줍니다. 누락된 권한은 단순히 해당 기능을 끌 뿐입니다 - 오류도, 망가진 패널도 없습니다.

| 권한 (PAT 범위)                 | 활성화하는 것                                                                                              | 없을 때                                                                                          |
|---------------------------------|------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------|
| **Code → Status**               | **Checks** 패널 - PR의 빌드/정책 결과.                                                                     | Checks를 로드할 수 없지만, PR 보기의 나머지 부분은 영향을 받지 않습니다.                          |
| **User Profile → Read**         | 어디서나 실제 프로필 사진.                                                                                  | 아바타가 이니셜로 렌더링됩니다.                                                                   |
| **Identity → Read**             | @-멘션 자동 완성, 멘션을 이름으로 해석하기, 그리고 기본 보기 뒤에 있는 팀 멤버십 조회.                       | 멘션 검색이 아무것도 반환하지 않으며, 팀에 할당된 PR이 기본 목록에서 사라질 수 있습니다.          |
| **Project and Team → Read**     | 기본 PR 목록의 **assigned to my team** 부분을 위한 팀 멤버십.                                              | 플러그인이 동일한 정보를 얻기 위해 Identity → Read로 대체합니다.                                  |
| **Work Items → Read**           | PR 세부 정보에 연결된 작업 항목과 Work Items 필터.                                                          | 작업 항목 섹션이 비어 있는 상태로 유지됩니다.                                                     |
| **Security → Manage**           | Complete 대화 상자의 **Override branch policies** 옵션.                                                     | 이 옵션이 숨겨집니다.                                                                             |

## 누락된 권한이 동작하는 방식

- **필수 권한 누락** - 로그인이 차단되며, 로그인 대화 상자에 정확한 범위 이름이 표시됩니다.
- **선택 권한 누락** - 해당 기능이 조용히 저하되고(사진 대신 이니셜, 빈 섹션) 나머지는 모두 계속 작동합니다.
- **팀에 할당된 PR을 사용할 수 없음** - 이는 기본 PR 목록에 표시되는 내용을 바꾸므로, 어떤 권한을 추가해야 하는지 설명하는 일회성 알림이 표시됩니다. 기본 "mine" 보기가 어떻게 작동하는지는 [풀 리퀘스트](Pull-Requests-ko.md)를 참조하세요.

## 누락된 권한 고치기

권한은 토큰 생성 시점에 새겨집니다 - 플러그인에서 기존 토큰에 추가할 수 없습니다.

<procedure title="더 많은 권한으로 다시 인증하기">
    <step><b>PAT:</b> 누락된 범위가 부여된 새 토큰을 만든 다음, <ui-path>Settings | Version Control | Azure DevOps</ui-path> → 연필 아이콘 → 붙여넣기 하세요.</step>
    <step><b>Microsoft (OAuth):</b> 계정을 제거하고 다시 로그인하면서 권한 선택기에서 <b>Full access</b>를 선택하세요.</step>
</procedure>

전체 로그인 안내는 [인증](Authentication-ko.md)을 참조하세요.
