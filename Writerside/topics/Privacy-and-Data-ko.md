# 개인정보 및 데이터

%product%를 사용할 때 어떤 데이터가 컴퓨터를 떠나는지, 어디로 가는지, 그리고 모든 것을 로컬에 유지하는 방법을 설명합니다.

## 자격 증명

| 자격 증명                 | 저장 위치                                                                                                            |
|--------------------------|---------------------------------------------------------------------------------------------------------------------|
| Azure DevOps PAT         | IDE의 `PasswordSafe` → 시스템 키체인(macOS Keychain, Windows Credential Manager, GNOME Keyring / KWallet).           |
| OAuth 갱신 토큰           | 동일 - `PasswordSafe`. 디스크에 평문으로 저장되는 경우는 없습니다.                                                    |
| AI 공급자 API 키          | 동일 - `PasswordSafe`, 공급자 인스턴스별로 키가 지정됩니다.                                                           |

OS별 키체인 세부 정보와 PAT를 교체하거나 취소하는 방법은 [인증](Authentication-ko.md)을 참조하세요.

## Azure DevOps로 전송되는 항목

이 플러그인은 Azure DevOps REST API 위에 있는 얇은 클라이언트입니다. 모든 호출은 사용자가 구성한 조직(`dev.azure.com/<org>` 또는 온프레미스 Azure DevOps Server)으로 직접 전송됩니다. 어떤 것도 서드파티 서버를 경유하지 않습니다.

호출은 다음과 같은 경우에 이루어집니다.

- PR 도구 창을 열 때(초기 목록 가져오기).
- 60초 백그라운드 동기화가 실행될 때.
- PR을 열 때(타임라인 + diff(변경 내용) 가져오기).
- 댓글 게시, 투표, 확인 표시, 완료 또는 중단을 할 때.
- Azure DevOps 원격 저장소에 대한 `git fetch` / `git push` 자격 증명 전달을 생성할 때.

## AI 공급자로 전송되는 항목 {id="whats-sent-to-ai-providers"}

AI가 **활성화**되어 있고 공급자가 구성된 경우에만 해당됩니다. 플러그인은 사용자가 트리거한 작업에 대해서만 아웃바운드 호출을 합니다.

> AI 공급자를 추가하거나 편집할 때, 플러그인은 모델 드롭다운을 채우기 위해 해당 공급자의 API 키를 사용하여 인증된 **모델 목록** 요청(예: `GET /v1/models`)을 한 번 보냅니다. 이것은 사용자가 트리거한 작업이 아니라 설정 시점에 실행되는 유일한 AI 요청입니다. PR 코드, diff(변경 내용), 프롬프트는 전송되지 않으며 - 모델 목록 호출만 전송됩니다 - 결과는 약 30분 동안 캐시됩니다. 로컬 공급자(localhost 주소의 Ollama)는 이를 사용자 컴퓨터에 유지합니다.
> {style="note"}

### 기능별 데이터 흐름

| 기능                    | 공급자가 보는 내용                                                                                              |
|------------------------|-----------------------------------------------------------------------------------------------------------------|
| **Summarize PR**       | PR 제목 + 설명 + diff(**Max diff size**로 잘림, 기본값 200 KB).                                                 |
| **AI review pass**     | 변경된 각 파일의 전체 파일별 diff. **Max diff size**보다 큰 파일은 건너뜁니다.                                   |
| **Explain code**       | 선택한 파일의 내용(보이는 범위만이 아니라 파일 전체).                                                            |
| **Commit message**     | 스테이징된 diff(`git diff --cached`가 생성하는 것과 동일).                                                       |
| **Title & description**| 브랜치 이름 + 커밋 메시지 + diff(위와 같이 잘림).                                                                |

각 AI 요청에는 플러그인이 해당 기능에 대해 구성한 시스템 프롬프트도 함께 전달됩니다. 이러한 프롬프트는 <ui-path>Settings | Version Control | Azure DevOps | AI Settings | Configure Prompts</ui-path>에서 재정의할 수 있습니다.

### 공급자 데이터 흐름 매트릭스

| 공급자                    | 요청이 전송되는 위치                                                                                                                            |
|--------------------------|------------------------------------------------------------------------------------------------------------------------------------------------|
| **Claude (Anthropic)**   | `api.anthropic.com`(또는 사용자 지정 기본 URL).                                                                                                |
| **OpenAI**               | 기본적으로 `api.openai.com`, 또는 사용자가 구성한 기본 URL(Azure OpenAI, vLLM, 자체 호스팅).                                                    |
| **Gemini (Google)**      | Google의 AI Studio / Vertex AI 엔드포인트.                                                                                                      |
| **Ollama**               | 사용자가 설정한 엔드포인트, 일반적으로 `http://localhost:11434`. localhost 주소인 경우 **네트워크 송신 없음**.                                   |
| **Claude Code CLI**      | `claude` 바이너리가 인증과 라우팅을 처리합니다 - 데이터 흐름은 Anthropic의 CLI 약관에 따라 제어됩니다.                                          |
| **OpenAI Codex CLI**     | `codex` 바이너리의 인증과 라우팅.                                                                                                               |
| **GitHub Copilot CLI**   | `copilot` 바이너리는 사용자의 Copilot 구독을 사용합니다 - 데이터 흐름은 GitHub Copilot의 약관에 따라 관리됩니다.                                |

플러그인은 이러한 요청에 헤더, 텔레메트리 또는 분석을 추가하지 않습니다. 업스트림 공급자가 보는 것은 정확히 플러그인이 전송한 것입니다.

## AI 응답 캐싱 {id="caching-ai-responses"}

기능이 결과를 반환하면, 플러그인은 다음을 키로 하여 결과를 캐시합니다.

- PR ID.
- 요청 시점의 diff SHA.
- 단조 증가하는 `cacheGeneration` 카운터 - 프롬프트를 편집하거나, 공급자를 변경하거나, 요약 카드의 톱니바퀴 팝업에서 **Clear cached AI responses**를 클릭할 때 증가합니다.

캐시 적중 시 아웃바운드 호출 없이 즉시 반환됩니다. 캐시된 응답은 IDE 로컬 상태에 저장되며 플러그인 제거 시 삭제됩니다.

## 텔레메트리

배포 버전의 플러그인은 **텔레메트리를 수집하지 않습니다**. 위에 나열된 API 호출(사용자의 Azure DevOps 조직 및 선택한 AI 공급자에 대한 호출) 외에는 사용 분석, 크래시 리포트 또는 "phone home" 핑이 없습니다.

향후 버전에서 옵트인 텔레메트리가 추가되는 경우, 기본적으로 꺼져 있으며 릴리스 노트에 명시됩니다.

## 모든 것을 로컬에 유지하기

데이터 레지던시 요구 사항이 있는 조직의 경우:

1. **온프레미스 Azure DevOps Server**(이전 TFS)를 사용하세요. PR, 댓글, API가 모두 자체 서버에 존재합니다.
2. 마스터 스위치로 **AI를 비활성화**하거나, `localhost`에서 실행되는 **Ollama 인스턴스로 모든 AI 기능을 라우팅**하세요. 오프라인 지원 모델(Llama, Mistral 등)과 함께 사용하면 어떤 데이터도 사용자 컴퓨터를 떠나지 않습니다.
3. 서드파티 CLI의 약관을 상속받고 싶지 않다면 **CLI AI 공급자를 피하세요** - 편리하지만 플러그인 입장에서 데이터 흐름이 불투명합니다.

## 사용자 지정 AI 공급자(엔터프라이즈) {id="custom-ai-providers-enterprise"}

%product%는 확장 지점을 노출하여 엔터프라이즈의 내부 플러그인이 기본 제공 AI 구현을 대체하고 예를 들어 내부 게이트웨이를 통해 AI 호출을 라우팅할 수 있도록 합니다. 확장 지점은 `plugin.xml`의 `intellij.vcs.azuredevops` 네임스페이스 아래에 선언되어 있습니다 - 구현 세부 정보는 플러그인의 GitHub 리포지토리를 참조하세요.

우선순위가 더 높은 확장이 등록되면, 해당 기능에 대해 플러그인의 기본 제공 기본값은 우회됩니다.
