# エディターでレビュー

diff ビューアーを使わずに、いつものコードエディター内で PR の変更を読んでコメントできます。チェックアウトしているブランチが PR のソースブランチと一致すると、エディター自体がレビューの場になります。

## 有効になるタイミング

エディターでのレビューは、次の**両方**が満たされると自動的に有効になります。

- プロジェクトに Azure DevOps のリモートがある。
- チェックアウトしているブランチが、オープンな PR の**ソースブランチ**と一致している。

すると、**メインツールバーの Git ブランチウィジェット**にその PR が表示され（例: `!1234 on feature/login`）、エディターに以下のレビュー機能が追加されます。

![エディター内で直接レビュー中の PR](review-in-editor.png){ width="720" border-effect="line" }

## エディターに表示されるもの

PR で変更のある各ファイルには、次のものが付きます。

- **ガター内の青い線** — 変更された範囲それぞれを示します。
- **ホバー時に表示される `+`** — ガター内でクリックするとその行にコメントでき、行番号をドラッグすると複数行の範囲を指定できます。
- **すでにスレッドがある行の控えめなハイライト** — そこに**スレッドのインレイ**がアンカーされます。スレッドのヘッダーをクリックすると展開または折りたたみができます。
- **エディター上部の `Review:` ツールバー** — **Prev Comment** / **Next Comment** でファイル内のスレッド間を移動できます。

スレッドは **PR の**行番号にアンカーされます。ファイルをローカルで編集しても、プラグインが変更を追跡し、各スレッドを正しい行に保ちます。

## コメントとレビュー

<procedure title="エディターからコメントを追加する">
    <step>ガター内の <b>+</b> をクリックします（またはキャレット位置で <shortcut>⌃⇧M</shortcut> / <shortcut>Ctrl+Shift+M</shortcut> を押します）。</step>
    <step>コメントを入力します。ほかの場所と同じツールバー、@メンション、提案、画像の貼り付けが使える完全な markdown です。</step>
    <step>すぐに投稿するか、保留中のレビューの一部としてキューに入れるか、提案された変更としてラップします。</step>
</procedure>

エディターから投稿したコメントと diff ビューアーから投稿したコメントは同一のコメントであり、どちらの画面でも完全なスレッドが表示されます。レビューを完了して投票するには、**Submit (N)** ボタンを使います（N はキューに入っているコメント数を表します）。ここで **Mark File as Viewed**（<shortcut>⌘⇧S</shortcut> / <shortcut>Ctrl+Shift+S</shortcut>）を行うこともでき、表示済みのファイルは変更ツリー内で淡色表示になります。

## オンとオフの切り替え

この切り替えは、**メインツールバーの Git ブランチウィジェット**のポップアップ内にあり、関連する 2 つのアクションと並んでいます。

![Review in Editor トグルのあるブランチウィジェットのポップアップ](branch-widget-popup.png){ width="440" border-effect="line" }

- **Show in Tool Window** — PR の詳細ビューを開きます。
- **Update Pull Request Branch** — ローカルブランチが PR のヘッドから分岐したときに表示されます。
- **Review in Editor** — ブランチを切り替えずに、エディターオーバーレイをオフにしたり再びオンにしたりします。

> 2 つのウィジェットを混同しないでください。**メインツールバー**のブランチウィジェットは `!{id} on {branch}` を表示し、これらのアクションを備えています。一方、別の**ステータスバー**ウィジェットは `ADO PR !{id}` を表示し、クリックすると単に PR を開きます。[Git 連携](Git-Integration-ja.md)を参照してください。
> {style="note"}

## ブランチ分岐時の自動オフ

ローカルブランチが PR のヘッドからずれると（通常はローカルでコミットしてまだプッシュしていない場合）、そのままでは古い行にスレッドをアンカーしてしまうため、エディターでのレビューは自動的にオフになります。PR のヘッドにプッシュまたはリセットするとすぐに再び有効になります。

## エディター vs diff ビューアー

エディターでのレビューは**追加的**なものであり、置き換えではありません。詳細ビューの変更ツリーから、いつでも PR の完全な diff を開けます。その時々に合ったほうを使ってください。

- **エディター** — import、ハイライト、いつものナビゲーションとともに、文脈の中でコードを読むのに最適です。
- **diff ビューアー** — 左右の並列比較や、変更されたすべてのファイルを一度に確認するのに最適です。

コメントは両者の間で同期されたままになります。
