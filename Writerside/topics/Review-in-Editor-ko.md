# 편집기에서 리뷰

diff 뷰어 없이도 일반 코드 편집기 안에서 PR의 변경 내용을 읽고 댓글을 달 수 있습니다. 체크아웃한 브랜치가 PR의 소스 브랜치와 일치하면 편집기 자체가 리뷰 화면이 됩니다.

## 언제 활성화되나요

편집기에서 리뷰는 다음 **두 조건**이 모두 참일 때 자동으로 켜집니다.

- 프로젝트에 Azure DevOps 원격이 있을 때.
- 체크아웃한 브랜치가 열린 PR의 **소스 브랜치**와 일치할 때.

그러면 **메인 도구 모음의 Git 브랜치 위젯**에 PR이 표시됩니다 - 예를 들어 `!1234 on feature/login` - 그리고 편집기에 아래의 리뷰 기능이 추가됩니다.

![편집기에서 바로 리뷰 중인 PR](review-in-editor.png){ width="720" border-effect="line" }

## 편집기에서 보이는 것

PR에서 변경된 내용이 있는 모든 파일에는 다음이 표시됩니다.

- **여백의 파란 선** - 변경된 각 범위를 표시합니다.
- **여백에 마우스를 올리면 나타나는 `+`** - 클릭하면 해당 줄에 댓글을 달 수 있고, 줄 번호를 가로질러 드래그하면 여러 줄 범위에 댓글을 달 수 있습니다.
- **이미 스레드가 있는 줄의 은은한 강조 표시** - **스레드 인레이**가 그 위치에 고정됩니다. 스레드 헤더를 클릭하면 펼치거나 접을 수 있습니다.
- 편집기 위의 **`Review:` 도구 모음** - **Prev Comment** / **Next Comment**로 파일 내 스레드 사이를 이동합니다.

스레드는 **PR의** 줄 번호에 고정됩니다. 파일을 로컬에서 편집하면 플러그인이 변경을 추적하여 각 스레드를 올바른 줄에 유지합니다.

## 댓글 달기와 리뷰

<procedure title="편집기에서 댓글 추가하기">
    <step>여백의 <b>+</b>를 클릭합니다(또는 캐럿 위치에서 <shortcut>⌃⇧M</shortcut> / <shortcut>Ctrl+Shift+M</shortcut>를 누릅니다).</step>
    <step>댓글을 입력합니다 - 전체 마크다운을 지원하며, 다른 곳과 동일한 도구 모음, @멘션, 제안, 이미지 붙여넣기를 사용할 수 있습니다.</step>
    <step>바로 게시하거나, 대기 중인 리뷰의 일부로 대기열에 넣거나, 제안된 변경으로 감쌀 수 있습니다.</step>
</procedure>

편집기에서 게시한 댓글과 diff 뷰어에서 게시한 댓글은 동일한 댓글입니다 - 두 화면 모두 전체 스레드를 보여줍니다. 리뷰를 마치고 투표하려면 **Submit (N)** 버튼을 사용합니다(대기 중인 댓글 수가 함께 표시됩니다). 여기서 **Mark File as Viewed**(<shortcut>⌘⇧S</shortcut> / <shortcut>Ctrl+Shift+S</shortcut>)를 사용할 수도 있으며, 확인한 파일은 변경 내용 트리에서 흐리게 표시됩니다.

## 켜고 끄기

이 토글은 **메인 도구 모음의 Git 브랜치 위젯** 팝업에 있으며, 관련된 두 가지 작업과 함께 있습니다.

![Review in Editor 토글이 있는 브랜치 위젯 팝업](branch-widget-popup.png){ width="440" border-effect="line" }

- **Show in Tool Window** - PR의 세부 정보 보기를 엽니다.
- **Update Pull Request Branch** - 로컬 브랜치가 PR 헤드에서 갈라졌을 때 나타납니다.
- **Review in Editor** - 브랜치를 전환하지 않고 편집기 오버레이를 끄거나 다시 켭니다.

> 두 위젯을 혼동하지 마세요. **메인 도구 모음** 브랜치 위젯은 `!{id} on {branch}`를 표시하며 이러한 작업을 제공합니다. 별도의 **상태 표시줄** 위젯은 `ADO PR !{id}`를 표시하며 클릭하면 단순히 PR을 엽니다. [Git 통합](Git-Integration-ko.md)을 참조하세요.
> {style="note"}

## 브랜치 갈라짐 시 자동 해제

로컬 브랜치가 PR의 헤드에서 갈라지면 - 보통은 로컬에서 커밋하고 아직 푸시하지 않았기 때문입니다 - 편집기에서 리뷰가 스스로 꺼집니다. 그렇지 않으면 스레드가 오래된 줄에 고정되기 때문입니다. PR 헤드로 푸시하거나 재설정하는 즉시 다시 활성화됩니다.

## 편집기 vs diff 뷰어

편집기에서 리뷰는 대체가 아니라 **부가적인** 기능입니다. 세부 정보 보기의 변경 내용 트리에서 언제든지 PR의 전체 diff(변경 내용)를 열 수 있습니다. 상황에 맞는 것을 사용하세요.

- **편집기** - import, 강조 표시, 평소의 탐색 기능과 함께 문맥 속에서 코드를 읽기에 가장 좋습니다.
- **diff 뷰어** - 나란히 비교하고 변경된 모든 파일을 한 번에 보기에 가장 좋습니다.

댓글은 두 화면 사이에서 동기화된 상태로 유지됩니다.
