# 在编辑器中审查

在你日常使用的代码编辑器中阅读并评论 PR 的改动——无需专门的差异查看器。当你检出的分支与某个 PR 的源分支匹配时，编辑器本身就成为审查界面。

## 何时激活

当以下**两个**条件都满足时，在编辑器中审查会自动开启：

- 你的项目有一个 Azure DevOps 远程仓库。
- 你检出的分支与某个开放 PR 的**源分支**匹配。

此时**主工具栏中的 Git 分支小组件**会显示该 PR——例如 `!1234 on feature/login`——并且编辑器会获得下文所述的审查功能。

![直接在编辑器中审查的一个 PR](review-in-editor.png){ width="720" border-effect="line" }

## 你在编辑器中看到的内容

PR 中每个有改动的文件都会获得：

- **装订线中的一条蓝线**，标记每个已更改的范围。
- **悬停时出现的 `+`**，位于装订线中——点击它可对该行发表评论，或跨行号拖动以选取多行范围。
- **已有讨论串的行上会有一处淡淡的高亮**，并在此处锚定**讨论串内嵌视图（thread inlays）**。点击讨论串标题可展开或折叠它。
- 编辑器上方的 **`Review:` 工具栏**，带有 **Prev Comment** / **Next Comment**，可在文件内的各讨论串之间跳转。

讨论串锚定到 **PR 的**行号。如果你在本地编辑该文件，插件会跟踪改动，并使每个讨论串保持在正确的行上。

## 评论与审查

<procedure title="从编辑器添加评论">
    <step>点击装订线中的 <b>+</b>（或在光标处按 <shortcut>⌃⇧M</shortcut> / <shortcut>Ctrl+Shift+M</shortcut>）。</step>
    <step>输入你的评论——支持完整的 markdown，配有与其他地方相同的工具栏、@提及、建议改动以及图片粘贴。</step>
    <step>立即发布它、将它排入一次待提交的审查队列，或将它包装为一项建议改动（suggested change）。</step>
</procedure>

从编辑器发布的评论与从差异查看器发布的评论是同一批评论——两个界面都会显示完整的讨论串。要完成一次审查并投票，请使用 **Submit (N)** 按钮（其中 N 是任何已排队评论的数量）。你也可以在这里 **Mark File as Viewed**（<shortcut>⌘⇧S</shortcut> / <shortcut>Ctrl+Shift+S</shortcut>），已查看的文件会在改动树中变暗。

## 打开或关闭它

该开关位于**主工具栏 Git 分支小组件**的弹出菜单中，与两个相关操作并列：

![带有 Review in Editor 开关的分支小组件弹出菜单](branch-widget-popup.png){ width="440" border-effect="line" }

- **Show in Tool Window**——打开 PR 的详情视图。
- **Update Pull Request Branch**——当你的本地分支与 PR head 出现分叉时出现。
- **Review in Editor**——无需切换分支即可关闭或重新开启编辑器叠加层。

> 不要混淆这两个小组件：**主工具栏**分支小组件显示 `!{id} on {branch}` 并承载这些操作；另一个独立的**状态栏**小组件显示 `ADO PR !{id}`，点击时只是打开该 PR。参见 [Git 集成](Git-Integration-zh.md)。
> {style="note"}

## 分支分叉时自动关闭

如果你的本地分支偏离了 PR 的 head——通常是因为你在本地提交而尚未推送——在编辑器中审查会自动关闭自身，否则它会把讨论串锚定到过时的行上。一旦你推送或重置到 PR head，它就会重新启用。

## 编辑器 vs 差异查看器

在编辑器中审查是**附加**功能，而非替代品。你随时可以从详情视图的改动树中打开 PR 的完整差异。根据当下需要选用其一：

- **编辑器**——最适合在上下文中阅读代码，带有 import、语法高亮以及你惯用的导航。
- **差异查看器**——最适合并排对比，并一次性查看每个已更改的文件。

评论会在两者之间保持同步。
